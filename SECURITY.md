## Supported Versions

Security fixes are applied to the default branch and latest release line.

## Reporting a Vulnerability

Do not open public issues for security vulnerabilities.

Report vulnerabilities through GitHub Security Advisories:

1. Open the repository Security tab.
1. Select Report a vulnerability.
1. Provide impact, reproduction details, and affected paths.

If GitHub Advisories are unavailable, contact the maintainer directly and include:

* Vulnerability description
* Reproduction steps
* Affected components and versions
* Suggested mitigation, if known

## Response Targets

The project aims for these response windows:

* Initial triage acknowledgment within 3 business days
* Impact assessment and severity classification within 7 business days
* Remediation plan or mitigation guidance as soon as practical

## Disclosure Process

Please allow time for triage and remediation before public disclosure.

Coordinated disclosure helps protect users while fixes are prepared and validated.
